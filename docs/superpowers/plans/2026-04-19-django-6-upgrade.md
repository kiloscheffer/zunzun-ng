# Django 5.2 → 6.0 + Python 3.11 → 3.14 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump pinned Django from `>=5.2,<5.3` to `>=6.0,<6.1` and pinned Python from `>=3.11` to `>=3.14`, keeping every other pattern from the prior 2.2 → 5.2 migration (JSON session storage, django-ratelimit, re_path URLs, spawn-based LRP, 82 pytest tests, 8-scenario smoke).

**Architecture:** Two separate version bumps in sequence, each with its own commit: Python first (isolates any Python-specific regression), Django second. Documentation sweep after both are green. Local merge only — no push to origin.

**Tech Stack:** Python 3.14.4 (target), Django 6.0.x (target), multiprocessing (spawn context, unchanged), django-ratelimit, pytest + pytest-django, uv for dependency management.

**Reference:** Design spec at `docs/superpowers/specs/2026-04-19-django-6-upgrade-design.md` (commit `08ecebd`). The spec covers motivation (stay-current, accepting STS support window), risk analysis, and non-goals.

## Global conventions

- **Test command:** `UV_LINK_MODE=copy uv run pytest tests/ -v`
- **Smoke command:** `UV_LINK_MODE=copy uv run python scripts/smoke_test.py` (~15 min on Windows)
- **Django check:** `UV_LINK_MODE=copy uv run python manage.py check`
- **All `uv` invocations need `UV_LINK_MODE=copy`** because the repo lives under Dropbox; hardlinking from uv's cache fails with Windows cloud-filesystem errors otherwise.
- **Commit style:** short subject matching repo style (no conventional-commit prefixes), body with `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` trailer.
- **Hook awareness:** `.claude/hooks/py_compile_check.py` runs on every `.py` Edit/Write; it blocks commits introducing syntax errors.
- **Working directory:** All phases execute inside `C:\Dropbox\git\zunzunsite3-django6\` (the worktree created in Task 1).
- **Fresh worktree needs `migrate`:** `session_db/db.sqlite3` is gitignored; run `UV_LINK_MODE=copy uv run python manage.py migrate` once before the first smoke invocation in the worktree, or smoke fails with `no such table: django_session`.
